## Message Correlation — the Messaging Triad

Every message in UTLXe carries three UUIDv7 identifiers:

| ID | Stays same? | Set by |
|----|:-----------:|--------|
| `MessageId` | No | Caller sets, or UTLXe generates UUIDv7 |
| `CorrelationId` | Yes | Preserved from origin — constant across the entire business process |
| `CausationId` | No | UTLXe sets to input's `MessageId` — links to immediate parent |

UTLXe always generates a new `MessageId` (UUIDv7) for the output and sets `CausationId` = input's `MessageId`. The caller only needs to pass the triad in — UTLXe handles the chain.

```
Input:   MessageId=UUID-A  CorrelationId=UUID-CORR  CausationId=UUID-PREV
Output:  MessageId=UUID-B  CorrelationId=UUID-CORR  CausationId=UUID-A
                    ^^^^                                        ^^^^^^
                    new UUIDv7                                  = input's MessageId
```

### Log output

Every message is logged with all three IDs:

```
INFO  [orders-in] MessageId=UUID-A CorrelationId=UUID-CORR CausationId=UUID-PREV
      → transformed in 3ms → output MessageId=UUID-B CausationId=UUID-A
```

Errors include the same IDs for correlation:

```
ERROR [orders-in] MessageId=UUID-A CorrelationId=UUID-CORR CausationId=UUID-PREV
      → FAILED at line 14: Null reference: $input.customer.address
```

## Example 1: Single Transformation (as it was, plus tracing)

The simplest case — one transformation, one input, one output. This is how it worked before, with the messaging triad added.

### C#

```csharp
// Load transformation (once at startup)
await client.LoadTransformationAsync(new LoadTransformationRequest
{
    TransformationId = "order-to-invoice",
    UtlxSource = File.ReadAllText("order-to-invoice.utlx"),
    Strategy = "COMPILED"
});

// Execute — single transformation
var response = await client.ExecuteAsync(new ExecuteRequest
{
    TransformationId = "order-to-invoice",
    Payload = ByteString.CopyFromUtf8(orderJson),
    ContentType = "application/json",

    // Messaging triad (all UUIDv7)
    MessageId = Guid.CreateVersion7().ToString(),   // UUID-A
    CorrelationId = correlationId,                  // UUID-CORR (from upstream)
    CausationId = upstreamMessageId,                // UUID-PREV (what caused this)
});

// Response:
//   response.MessageId     = UUID-B  (new UUIDv7, generated by UTLXe)
//   response.CorrelationId = UUID-CORR  (same — echoed)
//   response.CausationId   = UUID-A  (= our MessageId)
//   response.Output        = transformed payload
```

### Go

```go
resp, err := client.Execute(ctx, &utlxe.ExecuteRequest{
    TransformationId: "order-to-invoice",
    Payload:          orderPayload,
    ContentType:      "application/json",
    MessageId:        uuid.Must(uuid.NewV7()).String(),
    CorrelationId:    correlationId,
    CausationId:      upstreamMessageId,
})
// resp.MessageId     = new UUIDv7
// resp.CorrelationId = same
// resp.CausationId   = our MessageId
```

### UTLXe log

```
INFO  [order-to-invoice] MessageId=UUID-A CorrelationId=UUID-CORR
      → transformed in 2ms → output MessageId=UUID-B
```

## Example 2: Multi-Step Pipeline (no additional inputs)

A pipeline chains transformations: output of step 1 → input of step 2 → input of step 3. No additional inputs — just like before, with tracing.

### C#

```csharp
// Load three transformations
await client.LoadTransformationAsync(new LoadTransformationRequest
{
    TransformationId = "parse-order",
    UtlxSource = File.ReadAllText("parse-order.utlx"),
    Strategy = "COMPILED"
});
await client.LoadTransformationAsync(new LoadTransformationRequest
{
    TransformationId = "validate-order",
    UtlxSource = File.ReadAllText("validate-order.utlx"),
    Strategy = "COMPILED"
});
await client.LoadTransformationAsync(new LoadTransformationRequest
{
    TransformationId = "format-invoice",
    UtlxSource = File.ReadAllText("format-invoice.utlx"),
    Strategy = "COMPILED"
});

// Execute pipeline — just transformation names, no additional inputs
var response = await client.ExecutePipelineAsync(new ExecutePipelineRequest
{
    TransformationIds = { "parse-order", "validate-order", "format-invoice" },
    Payload = ByteString.CopyFromUtf8(rawOrderXml),
    ContentType = "application/xml",

    // Messaging triad for the pipeline
    MessageId = Guid.CreateVersion7().ToString(),   // UUID-A
    CorrelationId = correlationId,                  // UUID-CORR
    CausationId = upstreamMessageId,                // UUID-PREV
});

// No stage_inputs — simple pipeline, same as before

// Response:
//   response.MessageId      = UUID-B  (new UUIDv7 for the final output)
//   response.CorrelationId  = UUID-CORR
//   response.CausationId    = UUID-A
//   response.StagesCompleted = 3
//   response.Output         = final invoice XML
```

### Go

```go
resp, err := client.ExecutePipeline(ctx, &utlxe.ExecutePipelineRequest{
    TransformationIds: []string{"parse-order", "validate-order", "format-invoice"},
    Payload:           rawOrderXml,
    ContentType:       "application/xml",
    MessageId:         uuid.Must(uuid.NewV7()).String(),
    CorrelationId:     correlationId,
    CausationId:       upstreamMessageId,
})
// resp.StagesCompleted = 3
// resp.MessageId = new UUIDv7 for the final output
```

### UTLXe log (one entry per stage)

```
INFO  [parse-order]    MessageId=UUID-A CorrelationId=UUID-CORR stage=1/3 → 1ms
INFO  [validate-order] MessageId=UUID-A CorrelationId=UUID-CORR stage=2/3 → 1ms
INFO  [format-invoice] MessageId=UUID-A CorrelationId=UUID-CORR stage=3/3 → 2ms
      → pipeline complete in 4ms → output MessageId=UUID-B
```

## Example 3: Multi-Step Pipeline with Additional Inputs (EF01)

Same pipeline, but step 2 needs a customer table and step 3 needs a price list. The `stage_inputs` map provides these per stage, keyed by transformation name.

### C#

```csharp
// The customer table and price list — loaded from a file, API, or database
var customerTableJson = File.ReadAllBytes("customers.json");
var priceListCsv = File.ReadAllBytes("pricelist.csv");

var response = await client.ExecutePipelineAsync(new ExecutePipelineRequest
{
    // Same transformation chain
    TransformationIds = { "parse-order", "enrich-order", "calculate-price" },
    Payload = ByteString.CopyFromUtf8(rawOrderJson),
    ContentType = "application/json",

    // Messaging triad
    MessageId = Guid.CreateVersion7().ToString(),   // UUID-A
    CorrelationId = correlationId,                  // UUID-CORR
    CausationId = upstreamMessageId,                // UUID-PREV

    // EF01: Additional inputs per stage, keyed by transformation_id
    StageInputs =
    {
        // "parse-order" needs no additional inputs — not in the map

        // "enrich-order" needs the customer table
        ["enrich-order"] = new PipelineStageInputs
        {
            AdditionalInputs =
            {
                ["customers"] = ByteString.CopyFrom(customerTableJson)
            }
        },

        // "calculate-price" needs the price list
        ["calculate-price"] = new PipelineStageInputs
        {
            AdditionalInputs =
            {
                ["pricelist"] = ByteString.CopyFrom(priceListCsv)
            }
        }
    }
});

// In the "enrich-order" transformation:
//   $input     = output of "parse-order" (the pipeline flow)
//   $customers = the customer table (from additional_inputs)
//
// In the "calculate-price" transformation:
//   $input     = output of "enrich-order" (the pipeline flow)
//   $pricelist = the price list (from additional_inputs)

// Response:
//   response.MessageId       = UUID-B
//   response.CorrelationId   = UUID-CORR
//   response.CausationId     = UUID-A
//   response.StagesCompleted = 3
//   response.Output          = final priced order
```

### Go

```go
resp, err := client.ExecutePipeline(ctx, &utlxe.ExecutePipelineRequest{
    TransformationIds: []string{"parse-order", "enrich-order", "calculate-price"},
    Payload:           rawOrderJson,
    ContentType:       "application/json",
    MessageId:         uuid.Must(uuid.NewV7()).String(),
    CorrelationId:     correlationId,
    CausationId:       upstreamMessageId,

    StageInputs: map[string]*utlxe.PipelineStageInputs{
        "enrich-order": {
            AdditionalInputs: map[string][]byte{
                "customers": customerTableJson,
            },
        },
        "calculate-price": {
            AdditionalInputs: map[string][]byte{
                "pricelist": priceListCsv,
            },
        },
    },
})
```

### The .utlx transformations

```
// enrich-order.utlx
%utlx 1.0
input json
input customers json     // ← additional input, injected by pipeline
output json
---
{
  ...$input,
  customerName: find($customers, (c) -> c.id == $input.customerId).name
}
```

```
// calculate-price.utlx
%utlx 1.0
input json
input pricelist csv {header: true}   // ← additional input, CSV format
output json
---
{
  ...$input,
  unitPrice: find($pricelist, (p) -> p.sku == $input.productSku).price,
  total: $input.quantity * find($pricelist, (p) -> p.sku == $input.productSku).price
}
```

### UTLXe log

```
INFO  [parse-order]     MessageId=UUID-A CorrelationId=UUID-CORR stage=1/3 → 1ms
INFO  [enrich-order]    MessageId=UUID-A CorrelationId=UUID-CORR stage=2/3 → 3ms
                        additional_inputs: [customers]
INFO  [calculate-price] MessageId=UUID-A CorrelationId=UUID-CORR stage=3/3 → 2ms
                        additional_inputs: [pricelist]
      → pipeline complete in 6ms → output MessageId=UUID-B
```

## Example 4: Message Correlation Across a Chain

Shows the full causation chain across multiple services. Service A produces an order, UTLXe transforms it, Service B processes the invoice.

### The chain

```
Service A (producer):
  MessageId = UUID-A
  CorrelationId = UUID-CORR
  CausationId = (none — this is the origin)
        ↓
UTLXe transforms order → invoice:
  Input:  MessageId=UUID-A  CorrelationId=UUID-CORR  CausationId=(none)
  Output: MessageId=UUID-B  CorrelationId=UUID-CORR  CausationId=UUID-A
        ↓
Service B receives invoice, processes receipt:
  Input:  MessageId=UUID-B  CorrelationId=UUID-CORR  CausationId=UUID-A
  Output: MessageId=UUID-C  CorrelationId=UUID-CORR  CausationId=UUID-B
```

Every step: `CorrelationId` stays `UUID-CORR`, `CausationId` points to the parent.

### C# — Service A sends to UTLXe

```csharp
var myMessageId = Guid.CreateVersion7().ToString();   // UUID-A
var correlationId = Guid.CreateVersion7().ToString();  // UUID-CORR (new saga)

var response = await client.ExecuteAsync(new ExecuteRequest
{
    TransformationId = "order-to-invoice",
    Payload = ByteString.CopyFromUtf8(orderJson),
    ContentType = "application/json",
    MessageId = myMessageId,           // UUID-A
    CorrelationId = correlationId,     // UUID-CORR
    CausationId = "",                  // empty — this is the origin
});

// UTLXe returns:
//   response.MessageId     = UUID-B  (new UUIDv7)
//   response.CorrelationId = UUID-CORR  (preserved)
//   response.CausationId   = UUID-A  (= our MessageId)
```

### C# — Service B receives UTLXe's output, processes further

```csharp
// Service B received the invoice (UUID-B) from a queue
// It wants to create a receipt — continuing the chain

var receiptId = Guid.CreateVersion7().ToString();  // UUID-C

// Forward the triad:
var receipt = new Receipt
{
    MessageId = receiptId,                         // UUID-C (new)
    CorrelationId = incomingCorrelationId,          // UUID-CORR (same)
    CausationId = incomingMessageId,                // UUID-B (the invoice caused this)
    // ... receipt data
};

// The full chain is now:
//   UUID-A → UUID-B → UUID-C
//   All share UUID-CORR
//   Query by CorrelationId to find all messages in the saga
//   Follow CausationId backwards to trace the cause
```

### Querying the chain

```
// Find all messages in the saga:
SELECT * FROM messages WHERE correlation_id = 'UUID-CORR' ORDER BY message_id
// UUIDv7 = time-ordered, so ORDER BY message_id gives chronological order

// Trace the cause of a failure:
// Start with the failed message (UUID-C)
// → CausationId = UUID-B (the invoice)
// → CausationId = UUID-A (the order)
// → CausationId = empty (the origin)
```

## Proto Reference (relevant messages)

```protobuf
message ExecuteRequest {
  string transformation_id = 1;
  bytes payload = 2;
  string content_type = 3;
  map<string, bytes> named_inputs = 4;
  string correlation_id = 5;      // preserved across the chain
  string message_id = 6;          // UUIDv7, caller sets or UTLXe generates
  string causation_id = 7;        // UUIDv7, parent's MessageId
  map<string, string> metadata = 8;
  string traceparent = 9;         // W3C Trace Context
}

message ExecuteResponse {
  bool success = 1;
  bytes output = 2;
  string output_content_type = 3;
  string error = 4;
  ErrorClass error_class = 5;
  repeated ValidationError validation_errors = 6;
  ExecuteMetrics metrics = 7;
  ErrorPhase error_phase = 8;
  string correlation_id = 9;     // echoed
  string message_id = 10;        // new UUIDv7 for the output
  string causation_id = 11;      // = input's MessageId
  map<string, string> metadata = 12;
}

message PipelineStageInputs {
  map<string, bytes> additional_inputs = 1;
}

message ExecutePipelineRequest {
  repeated string transformation_ids = 1;  // ordered list of transformation names
  bytes payload = 2;
  string content_type = 3;
  string correlation_id = 4;
  string message_id = 5;
  string causation_id = 6;
  map<string, string> metadata = 7;
  string traceparent = 8;
  map<string, PipelineStageInputs> stage_inputs = 9;  // EF01: additional inputs per stage
}

message ExecutePipelineResponse {
  bool success = 1;
  bytes output = 2;
  string output_content_type = 3;
  string error = 4;
  ErrorClass error_class = 5;
  ErrorPhase error_phase = 6;
  repeated ValidationError validation_errors = 7;
  string correlation_id = 8;
  int32 stages_completed = 9;
  int64 total_duration_us = 10;
  string message_id = 11;        // new UUIDv7 for the final output
  string causation_id = 12;      // last stage's input MessageId
  map<string, string> metadata = 13;
}
```

---

*All MessageIds generated by UTLXe are UUIDv7 (RFC 9562) — time-ordered with an embedded millisecond timestamp. Callers may send any valid UUID; UUIDv7 is recommended.*
